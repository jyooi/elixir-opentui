What is OpenTUI?

Original (TypeScript)

A TypeScript library for building terminal UIs.
Created by Anomaly for OpenCode and terminal.shop.

Uses Zig rendering + Yoga layout engine.
Supports React, SolidJS, and Core API.

"Currently in development and is not ready for production use"

Ours (Elixir)

A complete Elixir reimagination of the same concepts.
Built on OTP/GenServer with MVU architecture.

Pure Elixir layout engine + optional Zig NIF.
Native macro DSL — no framework adapters needed.

412 tests passing. Works out of the box.

Code: Building a UI

TypeScript (Imperative)
import { createCliRenderer, Box, Text }
  from "@opentui/core"

const renderer = await createCliRenderer()

renderer.root.add(Box(
  { flexDirection: "row", gap: 2 },
  Box(
    { flexGrow: 1, bg: "#1a1b26" },
    Text({ content: "Sidebar" })
  ),
  Box(
    { flexGrow: 3 },
    Text({ content: "Main" })
  )
))
Elixir (Macro DSL)
import ElixirOpentui.View

box direction: :row, gap: 2 do
  box flex_grow: 1,
      bg: Color.hex("#1a1b26") do
    text content: "Sidebar"
  end

  box flex_grow: 3 do
    text content: "Main"
  end
end

Compile-time validated. No string keys.

Code: Components

TypeScript (React pattern)
function Counter() {
  const [count, setCount] = useState(0)

  useKeyboard((key) => {
    if (key.name === "up")
      setCount(c => c + 1)
    if (key.name === "down")
      setCount(c => c - 1)
  })

  return <text>{count}</text>
}

Requires React reconciler package

Elixir (MVU Behaviour)
defmodule Counter do
  use ElixirOpentui.Component

  def init(_props), do: %{count: 0}

  def update(:inc, _, state),
    do: %{state | count: state.count + 1}
  def update(:dec, _, state),
    do: %{state | count: state.count - 1}

  def render(state) do
    import ElixirOpentui.View
    text content: "#{state.count}"
  end
end

Zero extra packages. Pure behaviour.

Why ElixirOpentui?

  • Zero runtime deps — no node_modules, no Bun, no system Zig
  • OTP supervision — crash recovery built into the platform
  • Single process model — all state in one GenServer, no race conditions
  • Pure Elixir layout — no C++ Yoga FFI, still ~50µs for 500 nodes
  • Macro DSL — compile-time checked, no runtime parsing overhead
  • Optional NIF — Zig rendering is opt-in, not required
  • Headless testing — built-in TestRenderer for ExUnit
  • Works today — 412 tests passing, all 6 phases complete

Getting Started

TypeScript Setup
# Install Zig compiler
brew install zig
# Install Bun runtime
curl -fsSL https://bun.sh/install | bash
# Install package
bun install @opentui/core
# (optional) Install React reconciler
bun install @opentui/react react
# Configure tsconfig.json...
# Build native bindings...
bun run build

5+ steps, 2 system tools, framework choice

Elixir Setup
# Add to mix.exs deps:
{:elixir_opentui, "~> 0.1.0"}

# Install
mix deps.get

# Run
iex -S mix

That's it. No Zig. No build step. Works immediately.
